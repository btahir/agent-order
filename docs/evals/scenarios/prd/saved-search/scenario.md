# Saved Search PRD

Plan a saved-search feature for a project management application used by mid-size product teams (10-50 users per workspace).

Problem context:

- Users currently re-construct the same filter combinations many times per week (status, owner, project, label).
- Power users have asked for a way to name and pin a filter set; casual users have asked for the team's filters to surface to them so they don't have to learn them.
- The existing filters are URL-driven; deep links work but are not memorable.
- The team wants the feature in the next release (8 weeks away).

Constraints:

- Must work on web, iOS, and Android. The mobile apps share an API but render their own UIs.
- Must respect existing project-level permissions; a saved search must not surface items the viewer can't already see.
- The team can spare two engineers and one designer for the duration.

Produce a complete PRD that engineering and design could review next week.
