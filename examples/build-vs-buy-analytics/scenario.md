# Scenario: Build vs Buy Internal Analytics

A five-person SaaS team needs better internal analytics for product usage, account health, and support triage.

They currently have:

- Postgres as the primary database
- Stripe for billing
- Segment events going to a warehouse
- one full-stack engineer available part-time
- a goal to improve customer-success workflows within four weeks

They are deciding whether to build an internal dashboard or buy a lightweight analytics/customer-health tool.

## Constraints

- Avoid a large data-platform project.
- Prefer reversible decisions.
- Keep implementation and maintenance burden low.
- Make the recommendation useful for a small team, not an enterprise data org.

## Expected Output

Recommend build, buy, or hybrid. Include tradeoffs, risks, decision criteria, and a first-week implementation plan.
